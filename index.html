System-Dokumentation & Befehle

Hier findest du eine vollständige Übersicht. Klicke auf einen Befehlsnamen, um ihn sofort zu kopieren.

/ticket User / Support Team

Erstellt manuell ein neues Support-Ticket für den ausführenden Benutzer. Dieser Befehl öffnet einen geschützten Kanal, der ausschließlich für den Ersteller und die zugewiesenen Teammitglieder sichtbar ist.

/ticket-setup Owner

Initialisiert das automatisierte Ticket-System in einem frei wählbaren Kanal. Der Bot sendet ein professionelles Embed-Interface mit interaktiven Buttons.

/close Support Team

Schließt ein aktuell geöffnetes Support-Ticket permanent, sperrt den Schreibzugriff und generiert ein lückenloses Transkript (Log-Protokoll).

/giveaway Owner

Startet ein zeitgesteuertes Gewinnspiel direkt im Server. Der Bot erstellt ein interaktives Embed mit einem Button, über den User teilnehmen können.

/clear Owner

Löscht eine präzise Anzahl von Nachrichten aus dem aktuellen Textkanal (bis zu 100 Nachrichten gleichzeitig), um Chats nach Spam-Attacken sofort zu bereinigen.

/dice User / All

Ein interaktiver Unterhaltungsbefehl, der einen virtuellen, sechsseitigen Würfel wirft. Das System generiert eine zufällige Zahl zwischen 1 und 6.

/serverinfo User / All

Ruft umfassende statistische Daten des aktuellen Discord-Servers ab, inklusive Mitgliederzahlen, Erstellungsdatum und Boost-Status.

/userinfo User / All

Zeigt tiefgehende Detail-Informationen über ein bestimmtes Servermitglied an, wie das Beitrittsdatum, Accountalter und zugewiesene Rollen.

/emojiid Owner

Extrahiert die einzigartige, versteckte API-Identifikationsnummer (ID) eines benutzerdefinierten Server-Emojis für Konfigurationszwecke.

/lock Owner

Sperrt den aktuellen Textkanal augenblicklich für die Standard-Rolle (@everyone) bei plötzlichen Spam-Wellen oder Raids.

/unlock Owner

Hebt die Kanalsperre auf und stellt die normalen Schreibberechtigungen für alle regulären Benutzer im Textkanal wieder her.

/kick Owner

Wirft ein Mitglied mit sofortiger Wirkung aus dem Discord-Server. Der betroffene Benutzer kann über einen gültigen Link wieder beitreten.

/ban Owner

Verbannt einen Benutzer permanent vom Server und blockiert die IP-Adresse, um ein Wiederbetreten mit Zweitkonten zu verhindern.

/warn Owner

Erteilt einem Benutzer eine formelle Verwarnung wegen Fehlverhaltens und speichert diese permanent in der Systemdatenbank.

/warns Owner

Fragt das digitale Sündenregister und alle aktiven Verwarnungen eines Spielers ab, um wiederholte Verstöße einzusehen.

/setup Owner

Das Herzstück der Bot-Konfiguration. Öffnet ein umfassendes Konfigurations-Menü zur Bindung von Log-Kanälen, Kategorien und Rollen.
